# ERP 项目任务清单（按模块逻辑闭环）

状态说明
- `[x]` 已完成：代码完成，并已通过自动化验证或明确的手工验证。
- `[-]` 进行中：已有基础实现，但业务闭环、测试、权限、数据一致性或真实环境验收仍有缺口。
- `[ ]` 未开始：尚未形成可用实现。

闭环判断口径
- **主数据闭环**：新增 / 编辑 / 启停用 / 查询 / 导入导出 / 权限可控。
- **业务流闭环**：上游输入 -> 状态流转 -> 下游联动 -> 异常回滚或拦截 -> 报表追溯。
- **数据账闭环**：业务单据、余额、流水、幂等键、来源单据可互相追踪且数量一致。
- **多端闭环**：Web 做后台管理；Desktop/Mobile 只做现场操作，不放系统管理。
- **验收闭环**：每个模块在自己的章节内同时维护开发项、测试项和真实环境验收项。
