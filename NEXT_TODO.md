# ERP 业务流转落地计划

更新时间：2026-06-27

当前前提：项目已在 ECS 部署成功，可以登录。下一阶段只做能真实走单、能改状态、能影响库存或金额、能被用户验收的业务流转。暂不写“供应商绩效”“经营智能分析”这类没有具体规则和动作的目标。

状态说明：
- `[ ]` 未开始
- `[-]` 进行中
- `[x]` 已完成

优先级说明：
- `P0`：没有它，业务主链路走不通。
- `P1`：主链路走通后，提高多人协同和对账效率。
- `P2`：后续扩展，先不占当前排期。

## 0. 当前阶段原则

1. 先流转，后分析：先让采购、库存、生产、销售单据能顺着状态走完，再做报表和评分。
2. 先单据，后自动化：先人工点按钮形成清晰单据，再做自动审批、MRP、同步任务。
3. 先台账，后财务总账：先把应收、应付、收款、付款做成可查台账，不做完整会计凭证。
4. 先 Web，后 Mobile/Desktop：先在 Web 走通业务，再把高频现场动作搬到扫码和本地打印。
5. 每个功能都必须回答：谁操作、在哪个页面、点什么、状态变成什么、库存/金额怎么变。

## 1. V0：线上最小业务样本

目标：在线上录入一套最小业务数据，用它贯穿后续所有验收。

- `[ ]` 建基础角色
  操作人：管理员。
  页面：系统管理。
  验收：创建采购员、仓库员、生产员、销售员、财务员 5 个账号；每个账号只能进入自己的业务菜单。

- `[ ]` 建仓库和库位
  操作人：仓库员。
  页面：库存管理 / 仓库、库位。
  验收：创建 1 个原料仓、1 个成品仓；每个仓库至少 1 个库位。

- `[ ]` 建供应商、客户、原料、产品和 BOM
  操作人：采购员、销售员、生产员。
  页面：原料、供应商、客户、产品、生产 BOM。
  验收：2 个供应商、2 个客户、3 个原料、2 个产品/SKU、1 套成品 BOM 可以被采购单、生产单、销售单引用。

- `[ ]` 记录线上验收单据号
  操作人：管理员。
  页面：不限。
  验收：后续每条业务流都记录采购单号、入库单号、工单号、销售单号、出库单号，方便复查。

阶段出口：有一套真实线上样本数据，可以开始走采购、库存、生产、销售。

## 2. V1：采购到入库流转

目标：采购员下单，仓库员收货入库，系统增加原料库存，并形成应付台账。

### 2.1 采购单创建

- `[-]` 采购单手工创建
  操作人：采购员。
  页面：采购管理 / 采购订单。
  流转：新建采购单 -> 选择供应商 -> 添加原料明细 -> 保存草稿。
  状态：`DRAFT`。
  验收：采购单详情能看到供应商、物料、数量、单价、金额、预计到货日期。

- `[-]` 采购单提交
  操作人：采购员。
  页面：采购订单详情。
  流转：点击提交。
  状态：`DRAFT` -> `PENDING_APPROVAL`。
  验收：提交后不能随意修改明细；需要改动时必须退回草稿或作废重建。

### 2.2 采购审批

- `[ ]` 采购单通过 / 驳回
  操作人：有采购审批权限的用户。
  页面：采购订单详情。
  流转：点击通过或驳回，填写意见。
  状态：`PENDING_APPROVAL` -> `APPROVED` 或 `REJECTED`。
  验收：只有 `APPROVED` 采购单可以收货；`REJECTED` 采购单不能收货。

说明：当前先做明确的人工审批按钮，不做复杂金额规则、黑名单规则、绩效评分。

### 2.3 采购收货入库

- `[ ]` 采购收货
  操作人：仓库员。
  页面：采购订单详情或库存入库页面。
  流转：选择采购单 -> 填写本次收货数量、仓库、库位、批次号 -> 确认收货。
  状态：采购单按数量变为 `PARTIAL_RECEIVED` 或 `RECEIVED`。
  库存影响：原料库存余额增加；生成库存入库流水。
  验收：采购单收货数量、库存余额、库存流水三处数量一致。

- `[ ]` 收货数量校验
  操作人：系统自动校验。
  页面：采购收货。
  规则：本次收货数量不能小于等于 0；累计收货不能超过采购数量，除非后续明确支持超收。
  验收：非法数量提交失败并给出明确提示。

### 2.4 采购退货

- `[ ]` 采购退货
  操作人：采购员或仓库员。
  页面：采购退货。
  流转：选择已入库采购单 -> 填写退货数量和原因 -> 确认退货。
  状态：生成采购退货单。
  库存影响：原料库存余额减少；生成库存出库流水。
  金额影响：减少采购应付金额。
  验收：退货数量不能超过已入库未退数量。

### 2.5 采购应付台账

- `[ ]` 采购应付生成
  操作人：系统根据收货和退货计算。
  页面：采购应付。
  规则：应付金额 = 已入库金额 - 已退货金额 - 已付款金额。
  状态：`UNPAID`、`PARTIAL_PAID`、`PAID`。
  验收：采购单完成入库后能在应付页面看到未付金额。

- `[ ]` 付款登记
  操作人：财务员。
  页面：采购应付详情。
  流转：点击登记付款 -> 输入付款金额、付款日期、备注。
  金额影响：已付金额增加，未付金额减少。
  验收：付款金额不能超过未付金额；付清后状态变为 `PAID`。

## 3. V2：库存基础流转

目标：仓库员可以处理日常入库、出库、调拨、盘点，库存余额和流水始终对得上。

### 3.1 手工入库

- `[ ]` 手工入库
  操作人：仓库员。
  页面：库存管理 / 入库单。
  流转：选择物料或 SKU -> 填写数量、仓库、库位、原因 -> 确认。
  库存影响：库存余额增加；生成入库流水。
  验收：入库单、余额、流水数量一致。

### 3.2 手工出库

- `[ ]` 手工出库
  操作人：仓库员。
  页面：库存管理 / 出库单。
  流转：选择物料或 SKU -> 填写数量、仓库、库位、原因 -> 确认。
  库存影响：库存余额减少；生成出库流水。
  验收：库存不足时不能出库；失败提示包含当前可用库存。

### 3.3 库存调拨

- `[ ]` 库存调拨
  操作人：仓库员。
  页面：库存管理 / 调拨单。
  流转：选择来源仓库/库位、目标仓库/库位、物料或 SKU、数量 -> 确认调拨。
  库存影响：来源库位减少，目标库位增加；生成调拨流水。
  验收：调拨前后总库存不变，库位库存变化正确。

### 3.4 库存盘点

- `[ ]` 盘点单创建
  操作人：仓库员。
  页面：库存管理 / 盘点。
  流转：选择仓库或库位 -> 生成盘点单 -> 填写实盘数量。
  状态：`DRAFT` -> `SUBMITTED`。
  验收：盘点单能列出账面数量、实盘数量、差异数量。

- `[ ]` 盘点差异确认
  操作人：仓库主管或管理员。
  页面：盘点详情。
  流转：审核盘点差异 -> 通过或驳回。
  状态：`SUBMITTED` -> `APPROVED` 或 `REJECTED`。
  库存影响：通过后按差异调整库存余额，并生成盘盈/盘亏流水。
  验收：驳回不改变库存；通过后余额等于实盘数量。

## 4. V3：生产工单流转

目标：生产员按 BOM 创建工单，仓库发料，生产报工，完工入库成品。

### 4.1 创建生产工单

- `[-]` 生产工单创建
  操作人：生产员。
  页面：生产管理 / 生产工单。
  流转：选择产品/SKU -> 选择 BOM -> 填写计划生产数量和计划日期 -> 保存。
  状态：`DRAFT`。
  验收：工单详情能看到计划产量和按 BOM 展开的需领原料数量。

- `[ ]` 工单开工
  操作人：生产员。
  页面：生产工单详情。
  流转：点击开工。
  状态：`DRAFT` -> `IN_PROGRESS`。
  验收：未开工工单不能报工；已开工工单可以领料和报工。

### 4.2 生产领料和退料

- `[ ]` 生产领料
  操作人：仓库员。
  页面：生产工单详情或生产领料页面。
  流转：选择工单 -> 按 BOM 带出应领数量 -> 填写实领数量、仓库、库位 -> 确认。
  库存影响：原料库存减少；生成生产领料流水。
  验收：库存不足不能领料；领料后工单能看到已领数量。

- `[ ]` 生产退料
  操作人：仓库员。
  页面：生产退料。
  流转：选择工单和已领原料 -> 填写退料数量 -> 确认。
  库存影响：原料库存增加；生成生产退料流水。
  验收：退料数量不能超过已领未退数量。

### 4.3 生产报工和完工入库

- `[ ]` 生产报工
  操作人：生产员。
  页面：生产报工。
  流转：选择工单 -> 填写本次合格数量、报废数量、备注 -> 提交。
  状态：生成报工记录。
  验收：累计合格数量不能超过计划数量，除非后续明确支持超产。

- `[ ]` 完工入库
  操作人：仓库员。
  页面：生产工单详情。
  流转：选择已报工合格数量 -> 填写成品仓库、库位、批次或序列号 -> 确认入库。
  状态：工单按入库数量变为 `PARTIAL_COMPLETED` 或 `COMPLETED`。
  库存影响：成品库存增加；生成完工入库流水。
  验收：完工入库数量不能超过已报工未入库数量。

说明：排产、MRP、绩效统计后移。当前先保证一张工单能从创建走到完工入库。

## 5. V4：销售到出库流转

目标：销售员下销售单，仓库员发货出库，系统减少成品库存，并形成应收台账。

### 5.1 销售单创建

- `[-]` 销售单创建
  操作人：销售员。
  页面：销售管理 / 销售订单。
  流转：选择客户 -> 添加产品/SKU、数量、单价 -> 保存草稿。
  状态：`DRAFT` 或当前已有待确认状态。
  验收：销售单详情能看到客户、商品、数量、单价、金额、预计发货日期。

- `[ ]` 销售单确认
  操作人：销售员或销售主管。
  页面：销售订单详情。
  流转：点击确认。
  状态：`DRAFT` -> `CONFIRMED`。
  验收：未确认销售单不能发货；确认后锁定关键明细。

### 5.2 发货复核和销售出库

- `[ ]` 发货单生成
  操作人：仓库员。
  页面：销售订单详情或发货管理。
  流转：选择已确认销售单 -> 生成发货单。
  状态：发货单 `PENDING_REVIEW`。
  验收：发货单明细来自销售单，不允许发未下单商品。

- `[ ]` 发货复核
  操作人：仓库员。
  页面：发货单详情。
  流转：填写实际发货数量、仓库、库位、批次或序列号 -> 确认发货。
  状态：销售单按发货数量变为 `PARTIAL_SHIPPED` 或 `SHIPPED`。
  库存影响：成品库存减少；生成销售出库流水。
  验收：库存不足不能发货；发货数量不能超过销售数量。

### 5.3 销售退货

- `[ ]` 销售退货
  操作人：销售员或仓库员。
  页面：销售退货。
  流转：选择已发货销售单 -> 填写退货数量、原因、入库仓库/库位 -> 确认退货。
  状态：生成销售退货单。
  库存影响：成品库存增加；生成销售退货入库流水。
  金额影响：减少销售应收金额。
  验收：退货数量不能超过已发货未退数量。

### 5.4 销售应收台账

- `[ ]` 销售应收生成
  操作人：系统根据发货和退货计算。
  页面：销售应收。
  规则：应收金额 = 已发货金额 - 已退货金额 - 已收款金额。
  状态：`UNPAID`、`PARTIAL_RECEIVED`、`RECEIVED`。
  验收：销售单发货后能在应收页面看到未收金额。

- `[ ]` 收款登记
  操作人：财务员。
  页面：销售应收详情。
  流转：点击登记收款 -> 输入收款金额、收款日期、备注。
  金额影响：已收金额增加，未收金额减少。
  验收：收款金额不能超过未收金额；收清后状态变为 `RECEIVED`。

## 6. V5：异常先做最小处理

目标：先让异常有记录、有处理结果，不做复杂自动流转。

- `[ ]` 采购异常记录
  场景：到货数量不对、价格不对、供应商送错货。
  操作人：采购员或仓库员。
  页面：采购异常。
  流转：选择采购单 -> 填写异常类型、说明、处理结果。
  验收：异常记录能关联采购单；能标记为已处理。

- `[ ]` 库存异常记录
  场景：库存不足、批次找不到、账实不符。
  操作人：仓库员。
  页面：库存异常或库存流水备注。
  流转：填写异常说明 -> 处理完成后关闭。
  验收：异常记录能关联库存单据或库存流水。

- `[ ]` 生产异常记录
  场景：缺料、报废、返工。
  操作人：生产员。
  页面：生产工单详情。
  流转：选择工单 -> 记录异常类型、数量、说明、处理结果。
  验收：异常记录能关联工单；不要求当前自动影响成本。

- `[ ]` 销售异常记录
  场景：缺货、错发、客户退货争议。
  操作人：销售员或仓库员。
  页面：销售异常。
  流转：选择销售单或发货单 -> 记录异常类型、说明、处理结果。
  验收：异常记录能关联销售单或发货单；能标记为已处理。

## 7. 当前不做或后移

以下内容先不进入近期交付，避免计划发散：

- 供应商绩效评分。
- MRP 自动运算。
- 复杂排产和产线负荷。
- 客户信用额度自动拦截。
- 老板 Dashboard 和多维经营分析。
- 电商真实平台授权、拉单、库存同步。
- Mobile 离线缓存。
- 完整财务总账、凭证、报税申报。
- AI 分析、智能预测。

这些不是不要，而是等基础单据流转稳定后再做。

## 8. 当前最小交付顺序

只按下面顺序推进：

1. 线上最小业务样本。
2. 采购单：草稿 -> 提交 -> 审批通过 -> 收货入库 -> 生成应付 -> 登记付款。
3. 库存单：手工入库、手工出库、调拨、盘点调整。
4. 生产单：创建工单 -> 开工 -> 领料 -> 报工 -> 完工入库。
5. 销售单：草稿 -> 确认 -> 发货复核 -> 销售出库 -> 生成应收 -> 登记收款。
6. 退货：采购退货减少应付和库存；销售退货增加库存并减少应收。
7. 异常：采购、库存、生产、销售异常先能记录和关闭。

## 9. 每个流转的验收标准

每条流转完成时必须满足：

- 有明确页面入口。
- 有明确操作角色。
- 有状态变化。
- 有数量或金额变化时，能看到变化前后。
- 涉及库存时，库存余额和库存流水必须一致。
- 涉及应收应付时，未收/未付金额必须算清楚。
- 不能超收、超发、超退、超付、超收款。
- 失败时必须提示原因。
- 单据详情能看到关联单据和操作记录。
