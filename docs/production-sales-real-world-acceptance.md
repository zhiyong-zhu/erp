# 生产到销售发货真实环境验收清单

目标：用真实页面操作跑通 `工单 -> 投产 -> 领料 -> 报工 -> 装箱 -> 完工入库 -> 成品库存 -> 销售订单 -> 发货单 -> 复核出库 -> 库存流水`，并记录每一步的数据账。

## 1. 验收前准备

- 后端已启动并完成 Liquibase 迁移，确认 `056_production_material_movement.sql` 已执行。
- Web 端已登录具备生产、库存、销售权限的账号。
- 已存在可生产并可销售的产品、SKU、包装规格。
- 已存在该产品对应的生产 BOM，BOM 原料有足够原料库存余额。
- 已存在销售客户。

## 2. 页面操作路径

| 步骤 | 页面 | 操作 | 期望结果 | 数据账记录 |
| --- | --- | --- | --- | --- |
| 1 | 生产管理 / 生产工单 | 创建或确认一个待投产工单 | 工单状态为草稿或已下达 | 记录工单号、产品、计划数量、BOM |
| 2 | 生产管理 / 生产执行 | 点击 `投产` | 工单进入生产中 | 记录投产时间 |
| 3 | 生产管理 / 生产执行 | 点击 `领料`，按 BOM 领料 | 出现生产领料单，库存单据号生成 | 记录领料单号、库存出库单号、原料数量 |
| 4 | 库存管理 / 库存流水 | 查询领料库存流水 | 原料出现 `PRODUCTION_PICK` 或生产领料来源流水 | 核对原料余额减少数量 = 领料数量 |
| 5 | 生产管理 / 生产执行 | 提交 `报工` | 良品 + 不良品 = 报工数，累计良品不超过计划 | 记录报工单号、良品、不良品 |
| 6 | 生产管理 / 生产执行 | 可选生成序列号，再点击 `装箱` | 生成箱码，序列号进入装箱状态 | 记录箱码、序列号、装箱数量 |
| 7 | 生产管理 / 生产执行 | 点击 `入库` | 工单关闭，成品库存增加 | 记录完工入库时间、成品库存变化 |
| 8 | 库存管理 / 库存流水 | 查询完工入库流水 | 出现 `PRODUCTION_IN` 流水 | 核对成品余额增加数量 = 完工入库数量 |
| 9 | 销售管理 / 销售订单 | 新建销售订单并确认 | 订单状态进入待发货，成品库存被预占 | 记录销售订单号、SKU、数量、预占数量 |
| 10 | 销售管理 / 销售订单 | 点击 `发货`，可填写序列号 | 生成待复核发货单 | 记录发货单、运单号、发货数量 |
| 11 | 销售管理 / 发货管理 | 打开发货明细，点击 `复核出库` | 发货单变为已发货，销售订单变为已发货或部分发货 | 记录复核时间、发货状态 |
| 12 | 库存管理 / 库存流水 | 查询销售出库流水 | 出现 `SALE_OUT` 流水 | 核对成品余额减少数量 = 复核发货数量 |

## 3. 数量一致性口径

- 原料：领料后原料库存减少，退料后原料库存回补，流水来源可追溯到生产领退料单。
- 工单：累计良品数量不能超过计划数量，累计装箱数量不能超过累计良品数量。
- 完工入库：成品库存增加数量必须等于工单完工入库数量。
- 销售预占：确认或发货前预占成品库存，可用库存 = 当前库存 - 预占库存。
- 发货复核：复核后当前库存和预占库存同时减少，库存流水 `SALE_OUT` 数量为负数。
- 序列号：如果使用序列号，状态应按 `GENERATED -> PACKED -> STOCKED -> SHIPPED` 流转。

## 4. 异常验收

- 原料库存不足时，投产或领料应被拦截。
- 领料数量超过 BOM 计划需求时，应被拦截。
- 退料数量超过已领未退净量时，应被拦截。
- 报工良品 + 不良品不等于报工数时，应被拦截。
- 装箱数量超过累计良品时，应被拦截。
- 销售发货数量超过未发货数量时，应被拦截。
- 发货复核时成品库存或预占库存不足，应生成销售异常并阻止出库。
