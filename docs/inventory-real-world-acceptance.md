# 库存全账真实环境验收清单

目标：用真实环境跑通 `手工入库 -> 手工出库 -> 调拨 -> 盘点 -> 库存余额 -> 库存流水 -> 原料总库存`，并确认库存单据、余额和流水三者可互相追溯。

## 1. 验收前准备

- 后端已启动并完成 Liquibase 迁移。
- Web 端已登录具备原料、仓库、库位、入库、出库、调拨、盘点和库存流水权限的账号。
- 新建一条独立原料、一个独立仓库、两个独立库位，避免历史库存影响验收结果。

## 2. 页面操作路径

| 步骤 | 页面 | 操作 | 期望结果 | 数据账记录 |
| --- | --- | --- | --- | --- |
| 1 | 原料管理 / 原料列表 | 新建验收专用原料 | 原料初始库存为 `0` | 记录原料编码、原料 ID |
| 2 | 库存管理 / 仓库管理 | 新建验收仓库 | 仓库状态启用 | 记录仓库编码、仓库 ID |
| 3 | 库存管理 / 库位管理 | 新建 A、B 两个库位 | 两个库位均可用于库存单据 | 记录库位编码、库位 ID |
| 4 | 库存管理 / 正式入库单 | 手工入库 `100` 到 A 库位 | A 库位余额 `0 -> 100` | 记录入库单 ID、幂等键、`MANUAL_IN` 流水 |
| 5 | 库存管理 / 出库管理 | 手工出库 `12` 从 A 库位 | A 库位余额 `100 -> 88` | 记录出库单号、`MANUAL_OUT` 流水 |
| 6 | 库存管理 / 调拨管理 | 从 A 库位调拨 `30` 到 B 库位 | A 库位 `88 -> 58`，B 库位 `0 -> 30` | 记录调拨单号、`TRANSFER_OUT` / `TRANSFER_IN` 流水 |
| 7 | 库存管理 / 盘点管理 | 对 B 库位盘点实际 `33` | 创建后进入 `FROZEN` 并冻结账面库存 | 记录盘点单号、账面数量、实盘数量、差异数量 |
| 8 | 库存管理 / 盘点管理 | 复核盘点单 | 状态进入 `REVIEWED` | 复核后仍不产生调账流水 |
| 9 | 库存管理 / 盘点管理 | 审批通过盘点单 | 状态进入 `APPROVED`，B 库位余额 `30 -> 33` | 记录 `CHECK_PROFIT +3` 流水 |
| 10 | 库存管理 / 库存余额 | 按原料、仓库、批次查询余额 | A 库位 `58`，B 库位 `33` | 核对库存余额总和 `91` |
| 11 | 原料管理 / 库存台账 | 查询原料总库存 | 原料 `currentStock = 91` | 核对总库存 = A + B |
| 12 | 库存管理 / 库存流水 | 查询该原料流水 | 五条流水完整追溯到单据 | 核对 `100 - 12 - 30 + 30 + 3 = 91` |

## 3. 数量一致性口径

- 手工入库：`MANUAL_IN` 数量为正，余额增加。
- 手工出库：`MANUAL_OUT` 数量为负，余额减少，库存不足时应被拦截。
- 调拨：同一调拨单必须生成一正一负两条流水，调出库位减少，调入库位增加，总库存不变。
- 盘点：创建盘点单先进入 `FROZEN` 并冻结账面库存；复核进入 `REVIEWED`；审批通过进入 `APPROVED` 后才释放冻结、生成 `CHECK_PROFIT` / `CHECK_LOSS` 并把余额调整到实盘数量；驳回进入 `REJECTED` 并释放冻结，不产生调账流水。
- 总库存：原料 `currentStock` 必须等于所有库位库存余额之和。
